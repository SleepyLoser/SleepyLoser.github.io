    AI摘要

    嘿，大家好！今天我们来聊聊如何在Unity中实现一个复杂的Avatar系统。你可能已经在很多游戏和应用中见过这些系统，任天堂的动森、喷喷，QQ中的超级QQ秀等等。这些系统都允许用户自定义虚拟角色，从换装到动作，再到手持道具和特效，功能丰富且互动性强。无论你是游戏开发者还是对虚拟角色系统感兴趣，这篇文章都会为你提供一些实用的思路和方法。我们会一步步拆解这个系统的结构和实现逻辑，帮助你在Unity中打造出色的虚拟角色体验。那我们开始吧！

      


    简单的Avatar系统

    让我们从最基础的Avatar系统开始，这个系统将实现基本的骨骼绑定和动画播放。我们会一步步搭建这个系统，并逐步讲解每个部分的实现。

    基础结构与逻辑

    首先，我们需要了解Avatar系统的基本结构和逻辑。一个最简单的Avatar系统通常包括以下几个部分：

    1.  模型导入与设置：导入已经绑定骨骼的服装及人体部位的3D模型，转换为Prefab，并绑定自定义脚本存储信息。

    2.  装扮系统：将不同Prefab组装成完整Avatar，包括素体、服装、头发等。

    3.  动作系统：使用Animator Controller管理角色动作。

    Unity中的骨骼和SkinnedMeshRenderer

    骨骼系统

    在Unity中，骨骼系统是通过一系列Transform组件来实现的。这些Transform组件通常被称为骨骼或关节，它们构成了角色的骨骼层次结构。

    ● Transform组件：每个骨骼都是一个Transform组件，表示一个关节的位置、旋转和缩放。骨骼通常被组织成一个层次结构，每个骨骼都有一个父骨骼和零个或多个子骨骼。

    ● 骨骼层次结构：角色的骨骼层次结构类似于一棵树，从根骨骼开始，每个子骨骼都是一个分支。这个结构允许骨骼之间具有层次关系，当一个父骨骼移动时，所有的子骨骼都会相应地移动。

    ● 骨骼动画：通过操纵骨骼的Transform组件，动画系统可以控制角色的关节运动，生成复杂的动画效果。骨骼动画的核心是关键帧动画，动画师在关键帧设置骨骼的位置和旋转，动画系统在关键帧之间插值生成平滑的运动。

           

    SkinnedMeshRenderer

    SkinnedMeshRenderer是Unity中用于渲染带骨骼动画的网格的组件。它将网格与骨骼绑定在一起，使得网格能够随骨骼的变形而变形。

    ● 网格绑定：SkinnedMeshRenderer组件将网格的顶点绑定到骨骼上。每个顶点可以绑定到一个或多个骨骼，顶点的位置由绑定骨骼的变换矩阵决定。

    ● 骨骼权重：当一个顶点绑定到多个骨骼时，每个骨骼都有一个权重值，表示该骨骼对顶点位置的影响程度。通过调整骨骼的权重，可以实现更自然的变形效果。

    ● 动画驱动：SkinnedMeshRenderer组件通过骨骼动画驱动网格的变形。当骨骼动画播放时，SkinnedMeshRenderer会根据骨骼的变换矩阵实时计算顶点的新位置，从而生成动态变形的网格。

    骨骼系统与SkinnedMeshRenderer在Avatar系统中的作用

    在Avatar系统中，骨骼系统和SkinnedMeshRenderer共同发挥作用，实现角色的动态表现：

    1.  换装：通过将不同的服装和配件（绑定了SkinnedMeshRenderer）添加到Avatar的骨骼系统中，我们可以实现角色的换装功能。新的服装和配件会自动跟随骨骼的动画变形。

    2.  挂件：类似于换装，挂件（如武器、道具）也可以绑定到骨骼系统中，使其随角色的动作进行变形和移动。

    3.  特效：骨骼动画可以驱动特效（如魔法效果、攻击特效），使特效的表现更加生动和贴合角色的动作。

    4.  复杂动画：通过骨骼系统的层次结构和SkinnedMeshRenderer的网格绑定，可以实现角色的复杂动画，包括表情变化、动作切换和角色交互。

    通过正确理解和应用骨骼系统和SkinnedMeshRenderer，我们可以构建出功能丰富、表现力强的Avatar系统，满足多种应用场景的需求。


    模型导入与设置

    导入3D模型

    在Unity中，我们通常使用FBX格式的3D模型。将你的角色模型导入Unity项目中。导入的模型需要包括已经绑定好的骨骼，以便后续动画的实现。

    1.  将FBX文件拖入Unity的Project窗口中，这样模型就被导入到项目中了。

    2.  可以在在Project窗口中找到导入的模型文件，并将其拖入场景中查看，并确保它们的骨骼结构和材质正确配置。

        

    转换为Prefab

    将导入的模型转换为Prefab。Prefab是Unity中一种非常强大的功能，它允许我们将一个预设好的物体存储起来，以便在场景中多次实例化和复用。使用Prefab可以大大提高开发效率和资源管理的便利性。

    1.  将导入的模型FBX从Project窗口拖入场景中，调整位置、旋转和缩放。

    2.  右键点击模型的根节点，选择“Prefab - Unpack Completely”以解包模型。

    3.  然后将设置好的模型从场景中拖回到Project窗口中创建Prefab。


    绑定自定义脚本

    为每个部件Prefab绑定一个自定义脚本，用以存储部件的信息，如部件类型、ID或名字等。这些信息在Avatar组装时非常有用，因为它们允许我们方便地识别和管理各个部件。

    选择刚创建的Prefab，在Inspector窗口中点击“Add Component”，搜索并添加AvatarPart脚本。配置脚本中的相关参数。

    下面是一个示例的AvatarPart脚本，它不仅存储了部件的基本信息，还负责骨骼信息和SkinnedMeshRenderer的管理。

    using System.Collections.Generic;
    using System.Linq;
    using UnityEngine;

    public class AvatarPart : MonoBehaviour
    {
        public string avatarId; // 用于标识Avatar的唯一ID
        public AvatarPartType avatarPartType; // 部件类型，例如头发、衣服等
        public List<string> drivenBoneNameList; // 被驱动的骨骼名称列表
        private List<SkinnedMeshRenderer> skinMeshList; // SkinnedMeshRenderer组件列表

        private void Awake()
        {
                mainSkinMeshList = new List<SkinnedMeshRenderer>(GetComponentsInChildren<SkinnedMeshRenderer>(true));
                BuildDrivenBones();
        }

        private void BuildDrivenBones()
        {
            if (mainSkinMeshList.Count > 0)
            {
                drivenBoneNameList = new List<string>();
                foreach (var skinnedMeshRenderer in mainSkinMeshList)
                {
                    var bones = skinnedMeshRenderer.bones;
                    foreach (var bone in bones)
                    {
                        if (!drivenBoneNameList.Contains(bone.name))
                        {
                            drivenBoneNameList.Add(bone.name);
                        }
                    }
                }
            }
        }
    }

    这个脚本主要负责以下功能：

    ● avatarId：用于唯一标识每个Avatar部件，方便在组装和管理过程中进行识别。

    ● avatarPartType：存储部件的类型，例如是头发、衣服还是其他配件。这有助于在组装Avatar时快速识别和分类不同的部件。

    ● drivenBoneNameList：存储由此部件驱动的骨骼名称列表。在初始化过程中，脚本会遍历主要的SkinnedMeshRenderer组件，收集并存储所有的骨骼名称。

    ● skinMeshList：存储此部件中所有的SkinnedMeshRenderer组件，方便在后续的操作中进行访问和管理。

    通过这种方式，我们可以方便地在Unity中识别和管理各个部件。

    装扮系统

    装扮系统是Avatar系统的重要组成部分，它允许我们将不同的部件组合成一个完整的Avatar，并能够根据需要更换这些部件。下面我们详细讲解从每个部件组装成一个完整Avatar的具体步骤，以及换装过程中的实际操作。

    组装Avatar

    在组装Avatar时，我们需要进行以下几个步骤：

    1.  获取部件列表：定义Avatar所需的各个部件，例如素体、头发、衣服等。这些部件通常是以Prefab的形式存储在项目中，可以通过Inspector中直接选择Prefab，或者通过输入一个ID的列表来动态加载部件。

    2.  加载Prefab：根据部件列表加载相应的Prefab，可以使用Resource加载或者直接在Inspector中引用Prefab。

    3.  创建基础骨架：使用一个基础骨架Prefab，这个Prefab只包含骨骼结构，没有网格。这是一个全量骨骼，用于方便地合并各个部件的骨骼，作为主骨骼来使用。这样做的目的是确保所有部件共享同一个骨骼系统，并且所有动作都能同步。这一步可有可无，没有的情况下用任意部件的骨骼树作为主骨骼即可。

    4.  生成部件Prefab的实例：实例化每个部件的Prefab，生成克隆对象。

    5.  合并骨骼并修改Mesh的骨骼绑定：将每个部件的骨骼合并到基础骨架中，并更新网格的骨骼绑定，使所有部件能够协调一致地进行动画播放。

    关键步骤：合并骨骼

    前四个步骤都是简单的加载资源或者是简单的创建实例。关键步骤是第五步的合并骨骼，其本质是将新部件的骨骼结构根据骨骼名字匹配的逻辑合并入当前的主骨骼结构中，具体有以下几步：

    1.  获取基础骨架中的所有子骨骼：首先，我们需要获取基础骨架中的所有子骨骼，并存储在一个字典中，以便后续查找和合并。可以直接递归的方法获取所有的子骨骼。

    private void GetAllChildBones(Transform bone, Dictionary<string, Transform> boneDict)
    {
        if (bone == null) return;

        // 将当前骨骼加入字典中
        boneDict[bone.name] = bone;

        // 递归获取所有子骨骼
        foreach (Transform child in bone)
        {
            GetAllChildBones(child, boneDict);
        }
    }

    2.  合并骨骼：递归遍历每个部件的骨骼，将其合并到基础骨架中。如果在基础骨架中已经存在同名的骨骼，则直接使用该骨骼；否则，创建一个新的骨骼，复制Transform属性，并按原先的层次结构将其添加到主骨架中的对应位置。示例代码如下：

    private void MergeBones(Transform source, Transform targetParent)
    {
        if (source == null) return;

        // 检查主骨架中是否已存在同名骨骼
        if (mainBones.TryGetValue(source.name, out var existingBone))
        {
            // 如果存在，使用已有骨骼作为父节点
            targetParent = existingBone;
        }
        else
        {
            // 如果不存在，创建新的骨骼
            var newBone = new GameObject(source.name).transform;
            newBone.localPosition = source.localPosition;
            newBone.localRotation = source.localRotation;
            newBone.localScale = source.localScale;
            newBone.SetParent(targetParent, false);

            // 将新骨骼加入字典中
            mainBones[newBone.name] = newBone;

            // 更新父节点
            targetParent = newBone;
        }

        // 递归合并子骨骼
        foreach (Transform child in source)
        {
            MergeBones(child, targetParent);
        }
    }

    3.  重新映射网格的骨骼绑定：对于每个部件的SkinnedMeshRenderer组件，更新其骨骼绑定，使其受主骨架的控制。示例代码如下：

    private void RemapRendererBones(List<SkinnedMeshRenderer> sourceSkinList)
    {
        foreach (var sourceSkin in sourceSkinList)
        {
            var sourceBoneList = sourceSkin.bones;
            var newBone = new Transform[sourceBoneList.Length];

            // 更新每个骨骼的绑定
            for (var i = 0; i < sourceBoneList.Length; i++)
            {
                var sBone = sourceBoneList[i];
                var sBoneName = sBone.name;

                if (mainBones.ContainsKey(sBoneName))
                {
                    var mainBoneTransform = mainBones[sBoneName];
                    mainBoneTransform.localRotation = sBone.localRotation;
                    mainBoneTransform.localPosition = sBone.localPosition;
                    mainBoneTransform.localScale = sBone.localScale;
                    newBone[i] = mainBoneTransform;
                }
                else
                {
                    Debug.LogError($"UnConnected Bone {sBoneName} in {sourceSkin.name}");
                }
            }

            // 更新SkinnedMeshRenderer的骨骼数组
            sourceSkin.bones = newBone;

            // 更新根骨骼
            if (sourceSkin.rootBone != null)
            {
                var rootBoneName = sourceSkin.rootBone.name;
                var rootBone = mainBones[rootBoneName];
                sourceSkin.rootBone = rootBone ? rootBone : skeletonRoot.transform;
            }
        }
    }

    RemapRendererBones 方法更新每个部件的SkinnedMeshRenderer组件的骨骼绑定，使其与基础骨架对齐。SkinnedMeshRenderer组件需要更新以下部分：

    ● bones：这是一个Transform数组，包含了所有与网格顶点相关联的骨骼。我们需要将这些骨骼替换为与主骨架中的相应骨骼。

    ● rootBone：这是网格的根骨骼。我们需要将其替换为主骨架中的相应骨骼。

    通过以上步骤，我们已经详细讲解了如何在Avatar组装系统中合并骨骼。这个过程确保了所有部件共享同一个骨骼系统，并能够协调一致地进行动画播放。通过获取基础骨架中的所有子骨骼、递归合并骨骼，以及重新映射网格的骨骼绑定，我们成功地将Avatar的各个部件组装起来，形成一个完整的、可动画的角色。

        

    Avatar换装

    在完成了Avatar的初步组装后，我们接下来要讨论的是如何动态地更换Avatar的部件。换装是实现Avatar自定义的重要功能，通过这一过程，用户可以自由地更换Avatar的某个或多个部件，例如头发、服装、饰品等，使Avatar更加多样化和个性化。

    具体的换装过程可以分为以下几个步骤：

    1.  加载新部件的Prefab：从项目资源中加载新的部件Prefab。

    2.  删除旧部件并清理骨骼：销毁旧部件实例，并从主骨骼中移除旧部件的不在基础骨骼中的驱动骨骼，确保骨骼的整洁和统一。

    3.  实例化新部件并将其合并到主骨骼中：实例化新的部件，将新部件的骨骼合并到主骨骼中。

    4.  更新部件列表：在Avatar的部件列表中更新新的部件信息，以便后续管理和操作。

    下面我们具体展开每一步的实现细节。

    删除旧部件并清理骨骼

    在销毁旧部件时，还需要清理其骨骼信息，从主骨骼中移除该部件的不在基础骨骼中的驱动骨骼。这一步骤是为了确保一些部件存在新骨骼时，不会因为换下它们而造成骨骼残留。

    为了更好地理解这一步，我们需要明确以下几点：

    ● 基础骨骼：这是Avatar的核心骨骼系统，通常包含Avatar所有可能的骨骼结构。

    ● 主骨骼：这是当前Avatar实例正在使用的骨骼系统，包括基础骨骼和已合并的部件骨骼。

    ● 部件中的骨骼：这是每个部件自身的骨骼系统，当部件被合并到主骨骼时，这些骨骼会与主骨骼中的相应骨骼进行匹配或添加。

    当移除一个部件时，需要确保清理掉该部件的额外的骨骼，避免在主骨骼中留下多余的骨骼。

    private void RemoveAvatarPart(AvatarPartType avatarPartType)
    {
        if (currentPartDict.ContainsKey(avatarPartType) && currentPartDict[avatarPartType].gameObject != null)
        {
            RemoveRelatedDrivenBones(currentPartDict[avatarPartType].drivenBoneNameList);
            Destroy(currentPartDict[avatarPartType].gameObject);
            currentPartDict.Remove(avatarPartType);
        }
    }

    private void RemoveRelatedDrivenBones(List<string> drivenBoneNameList)
    {
        if (drivenBoneNameList == null) return;

        foreach (var boneName in drivenBoneNameList)
        {
            if (!basicBones.Contains(boneName) && mainBones.Keys.Contains(boneName))
            {
                var boneTransform = mainBones[boneName];
                boneTransform.SetParent(null);
                Destroy(boneTransform.gameObject);
                mainBones.Remove(boneName);
            }
        }
    }
    实例化新部件并将其合并到主骨骼中

    实例化新的部件，并替换当前Avatar上的相应部件。将新部件的骨骼合并到主骨骼中，这一步的实现与Avatar组装中的步骤完全一样。可以参考之前讲解的合并骨骼和更新SkinnedMeshRenderer的逻辑来实现。

    通过以上步骤，我们可以灵活地创建和修改Avatar，使其具备丰富的自定义能力。换装过程简单而高效，通过加载新部件Prefab、删除旧部件并清理骨骼、实例化并替换旧部件以及更新部件列表，实现了Avatar的动态换装。

    值得一提的是，这些换装逻辑和我们之前讲解的组装Avatar的逻辑可以通过一个方法实现，一简化整体的逻辑。例如：

    private IEnumerator UpdateAvatarWithDressList(List<GameObject> prefabList,
        List<AvatarPartType> removeList = null)

        if (removeList != null)

    动作系统

    在Avatar系统中，动作系统是使角色动起来的关键部分。动作系统通常使用Unity的Animator Controller来管理角色的动画。我们会先介绍基本的Animator Controller使用方法，然后再讨论如何在Avatar系统中实现动态的动作替换和融合。

    使用Animator Controller

    Animator Controller是Unity中管理动画状态的工具。通过Animator Controller，我们可以定义各种动画状态及其切换条件，使角色能够在不同状态之间流畅过渡。

    1.  创建Animator Controller：在项目窗口中右键点击，选择Create -> Animator Controller。命名为AvatarAnimatorController。

    2.  添加动画状态：双击打开Animator Controller，在状态机窗口中右键点击，选择Create State -> Empty。将其命名为Idle。同样的方法创建其他状态，如Walk、Run等。

    3.  设置动画剪辑：选择状态，在Inspector中为其分配相应的动画剪辑。

    4.  添加转换条件：在状态之间添加转换条件，例如从Idle到Walk的转换条件可以是isWalking布尔参数。

    在设置动画剪辑时，我们需要注意动画剪辑与骨骼的关系。每个动画剪辑都会影响绑定在相应骨骼上的网格。因此，在更换Avatar的部件时，新的部件必须有与动画剪辑匹配的骨骼结构。

    动态动作替换

    在Avatar系统中，我们需要根据不同的Avatar部件动态替换动作。例如，当角色换上不同的武器时，其攻击动作也应该相应变化。这可以通过Override Animator Controller来实现。

    1.  创建Override Animator Controller：在项目窗口中右键点击，选择Create -> Animator Override Controller。命名为WeaponOverrideController。

    2.  指定基础Animator Controller：在Inspector中，将AvatarAnimatorController拖动到Controller字段。

    3.  替换动画剪辑：在Override列表中，为Attack状态指定新的动画剪辑。

    然而，随着Avatar系统的复杂性增加，手动创建并管理这些Override Animator Controller会变得非常麻烦和低效。例如，每次添加新的装备或动作，都需要手动创建一个Override Animator Controller，然后应用到角色上。这样做不仅费时费力，还容易出错。

    那么，我们如何能够更高效地管理这些变化呢？答案是通过代码在运行时动态创建和管理Override Animator Controller。通过这种方式，我们不仅能提高效率，还能保证系统的灵活性和可扩展性。

    简单实现

    下面是一个简单的代码示例，展示了如何在Avatar系统中使用Animator Controller和Override Animator Controller来管理角色的动作，并在运行时动态替换动画剪辑。

    public class AvatarAnimatorManager : MonoBehaviour
    {
        public Animator animator; // 角色的Animator组件
        public RuntimeAnimatorController animatorController; // 通过Inspector引用的基础Animator Controller

        private AnimatorOverrideController overrideController; // 动态替换的Override Controller

        void Start()
        {
            // 初始化Animator Controller
            animator = GetComponentInChildren<Animator>(true);
            overrideController = new AnimatorOverrideController(animatorController);
            animator.runtimeAnimatorController = overrideController;
        }

        public void SetOverrideAnimation(AnimationClip newClip, string clipName)
        {
            if (overrideController == null)
            {
                Debug.LogError("OverrideController is not initialized.");
                return;
            }

            // 创建一个新的覆盖列表
            var overrides = new List<KeyValuePair<AnimationClip, AnimationClip>>();
            overrideController.GetOverrides(overrides);

            // 替换指定的动画剪辑
            for (int i = 0; i < overrides.Count; i++)
            {
                if (overrides[i].Key.name == clipName)
                {
                    overrides[i] = new KeyValuePair<AnimationClip, AnimationClip>(overrides[i].Key, newClip);
                }
            }

            // 应用新的覆盖列表
            overrideController.ApplyOverrides(overrides);
        }

        public void PlayAnimation(string animationName)
        {
            animator.Play(animationName);
        }
    }

    在这个示例中，我们通过代码动态创建和管理AnimatorOverrideController，并在运行时替换指定的动画剪辑。这使我们可以灵活地在运行时更改Avatar的动画，以适应不同的部件或装备。

    场景道具和特效

    在复杂的Avatar系统中，场景道具和特效是丰富游戏体验的重要元素。它们不仅可以增加角色与环境的互动性，还能为游戏带来酷炫的视觉效果。让我们一起看看如何实现场景道具和特效的加载与管理，并确保它们与角色的动画同步。

    场景道具是指那些可以和角色互动的物体，比如椅子、门或者宝箱。这些道具通常是普通的Mesh，有时也会带有骨骼和动画。想象一下你的角色走到一把椅子前，然后优雅地坐下来，这需要椅子和角色之间的动作能够完美同步。

    特效是让游戏更酷炫的视觉效果，比如魔法光环、爆炸效果或烟雾。特效可以直接放在场景中的某个位置，也可以绑定到角色的骨骼上，比如角色手上发出的魔法光芒。

    在Unity中，场景道具和特效通常以Prefab的形式存在。场景道具的Prefab可以是一个简单的Mesh，比如一把椅子，或者是一个带有骨骼和动画的复杂对象，比如一扇可以打开的门。每个场景道具Prefab都可能包含以下组件：Transform（定义对象的位置、旋转和缩放）、Mesh Renderer（渲染对象的外观）、Animator（如果道具有动画，这个组件用于控制动画的播放）。

    特效的Prefab通常由VFX系统创建，比如Particle System或VFX Graph。每个特效Prefab可以包含以下组件：Transform（定义特效的位置、旋转和缩放）、Particle System或VFX Graph（定义特效的具体表现形式）。

    动态加载和附加场景道具和特效的具体实现可以使用Unity的Resources.Load方法从项目资源中加载Prefab，使用Instantiate方法实例化Prefab，并将实例附加到场景中的指定位置或角色的骨骼上。

    实现细节

    现在，让我们详细讲讲每一步怎么做。

    首先，我们需要加载Prefab并实例化。想象一下你从资源库里找到了一个椅子模型，然后你把它放到了场景中角色旁边。你还可以把这个椅子绑定到角色的某个骨骼上，比如手上，这样角色可以“拿”着椅子。

    接着，如果这个椅子有动画（比如可以旋转），你需要给它设置一个Animator Controller。你可以使用角色的Animator Controller，确保它们的动画同步。通过监听角色动画的事件，可以触发相应的场景道具和特效的动画。比如，当角色开始坐下时，椅子的动画也会同步播放。

    不过，我们在这部份可以简化一下，所有场景道具都直接创建在角色的(0, 0, 0)位置，这时候要想实现角色拿着椅子这种效果可以通过动画的配合。至于挂在角色的骨骼上的道具手持物等，我们将在下一节讨论挂件与手持物时详细说明。

    附加场景道具

    这是一个加载到场景道具Prefab之后附加场景道具的示例。这些道具会被放置在角色的根节点下的“PropGroup”下，并记录在一个列表中，方便管理。

    private void AddAnimationProp(List<GameObject> props)

    附加特效

    特效的加载和附加通过如下例子实现。特效可能会附加到角色的骨骼上，以确保其与角色的动作同步。

    private void AddAnimationEffect(List<GameObject> effects)
    {
        foreach (var effect in effects)
        {
            var effectGameObject = Instantiate(effect, avatarRoot, false);
            var effectController = effectGameObject.GetComponent<EffectController>();
            if (effectController != null)
            {
                var mountBoneTransform = avatarController.GetBoneTransform(effectController.mountPoint);
                if (mountBoneTransform != null)
                {
                    effectGameObject.transform.SetParent(mountBoneTransform, false);
                }
            }
            effectObjList.Add(effectGameObject);
        }

        effectChangeCoroutine = null;
    }

    在附加特效的过程中，我们使用了EffectController组件。这个组件主要有两个作用：一是保存特效需要添加到哪个骨骼下，二是提供同步特效的方法。

    以下是EffectController的实现示例：

     public class EffectController : MonoBehaviour
    {
        public string mountPoint; // 特效需要附加的骨骼名称
        public bool needAlignToAnimation = false; // 是否需要与动画同步

        private List<ParticleSystem> effectParticles;

        private void Awake()
        {
            effectParticles = new List<ParticleSystem>(GetComponentsInChildren<ParticleSystem>());
        }

        public void ReplayEffect()
        {
            if (!needAlignToAnimation) return;
            
            foreach (var effectParticle in effectParticles)
            {
                effectParticle.Stop();
                effectParticle.Play();
            }
        }
    }

    在这个类中，mountPoint保存特效需要附加的骨骼名称，needAlignToAnimation指示特效是否需要与动画同步。Awake方法初始化特效粒子系统列表，ReplayEffect方法用于在需要时重新播放特效。

    *另外有一个值得关注的点是，有时特效会需要附加在动态骨骼下，在实际附加特效的时候，可以使用协程Coroutine的方法先确保Avatar的组装已经完毕再进行附加。

    动画同步

    在一个Avatar系统中，我们需要确保角色与场景道具和特效能够完美互动。例如，当角色坐在椅子上时，椅子的动画需要与角色的动作同步；当角色施放魔法时，特效需要准确地显示在角色的手上。通过上述方法，我们可以实现这些互动和特效，并确保它们与角色的动作同步。但在附加时肯定无法确保他们之间的播放进度同步，所以需要额外在播放时进行同步：

    public void SyncPlayAnimation()
    {
        // 播放角色动画
        animator.Play("Anim", -1, 0);

        // 同步场景道具动画
        if (animPropList.Count != 0)
        {
            foreach (var prop in animPropList)
            {
                var propAnimator = prop.GetComponent<Animator>();
                if (propAnimator)
                {
                    propAnimator.speed = playAnimationSpeed;
                    propAnimator.Play("PlayAction", -1, 0);
                    propAnimator.Update(0);
                }
            }
        }

        // 同步特效
        if (effectObjList.Count > 0)
        {
            foreach (var effect in effectObjList)
            {
                if (effect == null)
                {
                    Debug.LogError("effect object list null");
                    continue;
                }
                var effectController = effect.GetComponent<EffectController>();
                effectController.ReplayEffect();
            }
        }
    }

    在这个方法中，我们首先播放角色的动画。接着，遍历animPropList同步每个场景道具的动画，确保它们与角色的动画速度一致，并在相同的时间点开始播放。然后，遍历effectObjList，通过调用EffectController的ReplayEffect方法，重新播放特效，确保特效与动画同步。


    挂件与手持物

    在Avatar系统中，挂件和手持物是角色个性化的重要元素。通过动态加载和附加这些物品，可以增强角色的表现力和互动性。这些物品能够显著增强角色的个性化表现，例如，角色可以佩戴不同风格的项链、耳环，或者持有各种武器、盾牌。这不仅增加了视觉上的多样性，还可以影响角色的行为和互动。例如，角色手持武器时可以进行战斗动作，佩戴特殊挂件时可以触发特定的技能效果。

    挂件和手持物通常会有额外的骨骼，这些骨骼用来控制它们的变形和动画。它们需要附加到角色骨骼系统中的特定位置，例如武器挂在手骨上，项链挂在脖子骨上。有时，挂件和手持物会有自己的动画和特效，例如，武器可能会有闪烁的光效，手持物可能会有挥舞动作。

    实现挂件和手持物的方法主要有两种：一种是独立骨骼和动画控制，另一种是骨骼合并和集中控制。

    独立骨骼和动画控制

    每个挂件和手持物有自己的骨骼和Animator Controller，控制自己的动画和特效。具体实现方法是加载并实例化挂件或手持物的Prefab并将其附加到指定的骨骼点上，然后挂件或手持物的动画和特效由自身的Animator Controller以及VFX控制。

    骨骼合并和集中控制

    另一种方法是将挂件和手持物的骨骼与角色的骨骼系统合并，通过Avatar的Animator Controller控制动画。具体实现方法是加载并实例化挂件或手持物的Prefab，将其骨骼合并到角色的骨骼系统中，然后在Avatar的Animator Controller中使用额外的动画层（Layer）来控制挂件或手持物的动画。

      

    两种方法的优劣

    独立骨骼和动画控制的方法实现简单，易于管理和调试，挂件和手持物的动画和特效独立于角色，修改和更新方便。但在复杂的场景中，多个Animator Controller可能会增加CPU和内存开销。

    骨骼合并和集中控制的方法性能较好，所有动画由一个Animator Controller统一控制，减少了计算开销，动画同步更精准。然而实现较复杂，需要处理骨骼合并和动画层的管理，调试难度较大。

    实际开发中的问题与注意事项

    在实际开发中，可能会遇到以下问题和注意事项：

    ● 性能问题：多个Animator Controller可能会增加CPU和内存开销，特别是在使用独立骨骼和动画控制方法时。需要优化动画资源，确保动画流畅。

    ● 动画同步：确保挂件和手持物的动画与角色动画同步，特别是在角色进行复杂动作（如跳跃、攻击）时，需要仔细调整动画的起始时间和速度。

    ● 骨骼合并的冲突：在骨骼合并时，可能会遇到骨骼名称冲突的问题。需要确保挂件和手持物的骨骼名称与角色的骨骼名称不冲突，或者在合并前进行重命名。

    ● 调试和测试：挂件和手持物的加载和绑定过程需要仔细调试，确保位置和旋转正确。在各种场景和动作中测试挂件和手持物的表现，确保它们在任何情况下都能正常工作。


    分层动画

    在Avatar系统中，分层动画是实现复杂动画行为的重要技术。通过使用Animator Controller的动画层（Layer），我们可以让角色的不同部分独立地播放动画，例如角色的上半身和下半身可以执行不同的动作。让我们来详细看看如何实现场景中的分层动画。

    分层动画的逻辑

    分层动画允许我们将角色的动画分解为多个独立的部分，并在运行时将它们组合在一起。这样，我们可以实现更灵活和复杂的动画行为。例如，角色在行走时可以同时挥舞武器，或者在奔跑时进行攻击。

    实现方法

    使用Animator Controller的动画层功能，我们可以为不同的身体部位设置独立的动画层。每个动画层可以有自己的状态机和动画逻辑，这样我们可以在同一时间播放多个动画，并且这些动画可以相互独立地控制。

    以下是实现分层动画的具体步骤和代码示例：

    创建动画层并配置Mask

    在Animator Controller中，添加新的动画层。例如，我们可以创建一个名为“Upper”的动画层，用于控制上半身的动画。接着为动画层设置遮罩，并设置blending属性为Override。

    ● 设置遮罩（Avatar Mask）：遮罩用于指定动画层影响的身体部位。通过使用遮罩，我们可以确保某个动画层只影响角色的特定部分。例如，可以创建一个遮罩，使上半身动画层只影响角色的上半身的骨骼，而不影响下半身的骨骼。

    ● 使用Override模式：Override模式确保动画层中的动画完全覆盖基础动画层的动画。例如，如果基础动画层正在播放角色的行走动画，而上半身层播放攻击动画，使用Override模式可以确保攻击动画不会受到行走动画的影响。

    配置状态机和动画剪辑

    在每个动画层中，配置相应的状态机和动画剪辑。例如，在上半身层中，可以设置挥舞武器的动画；在下半身层中，可以设置行走或奔跑的动画。

    // 设置Animator的动画层权重
    animator.SetLayerWeight(animator.GetLayerIndex("Upper"), 1.0f);
    animator.SetLayerWeight(animator.GetLayerIndex("Body"), 1.0f);

    // 播放上半身和全身的动画
    animator.Play("UpperAnim", animator.GetLayerIndex("Upper"), 0);
    animator.Play("BodyAnim", animator.GetLayerIndex("Body"), 0);

    在实际开发中，我们通常会为角色设置一个基础的全身动作，这个动作会覆盖角色的所有骨骼。当需要特定的动画效果时，我们可以使用额外的动画层来覆盖基础动作的部分区域。通过设置动画层的权重和遮罩，我们可以精确控制动画层的影响范围。

    例如，角色的基础动作是行走，当角色需要进行攻击动作时，我们可以为上半身设置一个动画层，并播放攻击动画。这样，上半身的攻击动作会覆盖基础的行走动作，而下半身依然保持行走状态。

    animator.Play("Walk", 0);

    public void PlayAttackAnimation()
    {
        animator.SetLayerWeight(animator.GetLayerIndex("Upper"), 1f);
        animator.Play("Attack", animator.GetLayerIndex("Upper"), 0);
    }

    动画层的优劣

    使用动画层有许多优点，包括灵活性和可控性。然而，也需要注意一些潜在的问题：

    ● 优点：动画层允许我们在同一时间播放多个动画，增强了动画系统的灵活性。通过使用遮罩，我们可以精确控制每个动画层影响的身体部位，实现复杂的动画行为。

    ● 缺点：动画层的管理和调试可能比较复杂，特别是在有多个动画层和复杂状态机的情况下。需要仔细调整每个动画层的权重和遮罩，确保它们之间不会产生冲突。

    过渡更加自然的实现

    直接让不同动画通过Layer控制不同的骨骼有时候会导致动作很僵硬不自然。这是因为身体不同部分的动画在过渡时，可能会缺乏平滑的连接，尤其是脖子、腰部等连接部位。为了让动画过渡更加自然，可以在连接处增加一个与部位layer同步， 但权重不是1的Override层。

    在Animator Controller中，为连接部位创建一个新的动画层，这个层的权重可以通过代码动态调整，从而实现更自然的过渡效果：

    // 动态调整连接部位的权重
    public void AdjustBlendLayerWeight(float weight)
    {
        int blendLayerIndex = animator.GetLayerIndex("BlendLayer");
        animator.SetLayerWeight(blendLayerIndex, weight);
    }

    // 在过渡时调用调整权重的方法
    public void TransitionWithBlend()
    {
        animator.Play("Run", animator.GetLayerIndex("Body"), 0);
        animator.Play("Attack", animator.GetLayerIndex("Upper"), 0);
        AdjustBlendLayerWeight(0.8f);
    }

    实际开发中的问题与注意事项

    在实际开发中，使用动画层时可能会遇到以下问题和注意事项：

    ● 动画层冲突：多个动画层可能会影响相同的身体部位，导致动画冲突。需要仔细设置每个动画层的遮罩，确保它们只影响指定的身体部位。

    ● 性能问题：在有多个动画层和复杂状态机的情况下，Animator Controller的性能可能会受到影响。需要优化动画资源和状态机，确保动画流畅。


    动态动作过渡

    在复杂的Avatar系统中，角色的状态动作往往会触发一组相关联的反应动作。这些动作之间需要自然过渡，才能确保角色的动画流畅和真实。在处理动态设置的多个状态动作以及这些动作之间的自然过渡时，这一功能尤为重要。

    虽然接下来要说的是实际项目中遇到的一个非常实际的需求，但通过实现这一功能，我们也可以学到如何管理多动作以及自定义过渡等方法。

    动态动作切换的逻辑

    在Avatar动画系统中，一个状态动作（如动作A）可能会触发一系列关联的反应动作（如A1 - A4）。为了实现这些动作的自然过渡，我们可以通过有限的3个state来动态管理并播放超过3个的状态动作。这种方法利用了Animator Controller中的Transition交叉过渡功能，使得状态之间的切换更加灵活和自然。

    具体的实现逻辑如下：

    1.  状态A播放时，将动画剪辑替换到Anim这个state中。

    2.  当需要切换到A1时，将a1替换到一个空的AdditionState1中，然后进行切换。

    3.  如果需要切回状态A，则切回Anim。

    4.  如果需要播放其他关联动作（如A2 - A4），则将目标动作替换到另一个空的AdditionState2中，然后进行切换。

    实现方法
    创建状态机

    在Animator Controller中，创建3个主要状态（State），每个状态对应一组动画。右键点击Animator Controller的空白区域，选择“Create State -> Empty”来创建一个新的状态。然后，拖动相应的动画剪辑到该状态中。例如，创建“Idle”、“Anim”、“AdditionState1”、“AdditionState2”等状态，并将相应的占位用动画剪辑分配给这些状态，用于之后可以通过override controller动态替换动画剪辑。

    设置转换条件和过渡参数

    使用参数（Parameter）和条件（Condition）来控制状态之间的转换。设置过渡参数，用于不同动作间的交叉过渡。

    在设置过渡参数时，可以调整以下参数：

    ● Has Exit Time：是否等待当前动画播放完毕。如果勾选，动画会在播放完毕后再进行过渡。

    ● Exit Time：当前动画需要播放的比例（0到1之间）。例如，设置为0.75表示动画播放到75%时开始过渡。

    ● Transition Duration：过渡的持续时间。例如，设置为0.15表示过渡会在0.15秒内完成。

    ● Transition Offset：过渡开始时新动画的偏移量。例如，设置为0表示从新动画的开始播放。

    替换动画剪辑并触发状态切换

    使用override controller替换动画剪辑，并通过设置的转换条件，可以实现多个状态动作之间的自然过渡。

    animator.SetTrigger("AdditionState1");
    使用CrossFade方法实现自然过渡

    在某些情况下，我们希望可以不触发特定参数直接过渡到目标状态，或者是希望使用与animtor中预设的过渡参数不同的设置。这时可以使用CrossFade方法，该方法可以在指定的时间内从当前状态平滑过渡到目标状态。

    以下是一个示例代码，展示了如何使用CrossFade方法来实现自然过渡：

    public void PlayDynamicActionDirectly(string actionName, float transitionDuration = 0.1f)
    {
        // 获取当前状态信息
        var stateInfo = animator.GetCurrentAnimatorStateInfo(0);

        // 动态创建Animator Override Controller
        var overrideController = new AnimatorOverrideController(animator.runtimeAnimatorController);
        overrideController["Anim"] = Resources.Load<AnimationClip>($"Animations/{actionName}");

        // 应用Override Controller
        animator.runtimeAnimatorController = overrideController;

        // 进行交叉过渡
        animator.CrossFadeInFixedTime("Anim", transitionDuration);
    }

    CrossFade 和 CrossFadeInFixedTime 是 Unity Animator 中用于实现动画平滑过渡的方法。以下是对每个参数的详细解释。

    CrossFade
    public void CrossFade(string stateName, float transitionDuration, int layer = -1, float normalizedTimeOffset = 0f, float normalizedTransitionTime = 0f);

    ● stateName：目标动画状态的名称。

    ● transitionDuration：过渡的持续时间，单位是秒。

    ● layer：动画层的索引。指定要在哪个动画层上执行过渡。默认值是-1，使用默认层。

    ● normalizedTimeOffset：目标状态的标准化时间偏移。目标状态开始播放时的标准化时间（0到1之间）。默认值是0，表示从动画的开始播放。

    ● normalizedTransitionTime：标准化的过渡时间。表示过渡时间的标准化值（0到1之间）。默认值是0，表示过渡时间相对于当前动画的结束时间。

    CrossFadeInFixedTime
    public void CrossFadeInFixedTime(string stateName, float fixedTransitionDuration, int layer = -1, float fixedTimeOffset = 0f);

    ● stateName：目标动画状态的名称。

    ● fixedTransitionDuration：固定的过渡持续时间。

    ● layer：动画层的索引。

    ● fixedTimeOffset：目标状态的固定时间偏移。

    通过合理使用这些方法这些参数，可以在复杂的动画系统中实现自然的动作切换和过渡。这些方法在处理动态动画时尤其有用，使得动画状态的管理更加灵活和高效。
